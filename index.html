installer SASS

Sur Ubuntu

Installer ruby :

sudo apt install ruby

Installer ruby-sass :

sudo apt-install ruby-sass

Ruby est nativement installé, il suffit donc d'installer SASS :

gem install sass

Utilisez ruby installer

Installer SASS :

gem install sass

Compiler un fichier .scss en .css

On crée un fichier style.scss, dans lequel on va écrire notre code en scss. Il est inutile de créer un fichier style.css : Sass s'en chargera !
Les navigateurs ne lisant pas les fichiers .scss, il nous faut le compiler en css, avec cette commande (qu'on lance depuis le dossier dans lequel on a créé notre style.scss) :

sass --watch style.scss:style.css

Important : C'est bien sur votre style.css et non style.scss que votre balise meta stylesheet doit pointer. Souvenez-vous que votre navigateur ne lit pas le scss !

La syntaxe SCSS

L’une des caractéristiques les plus intéressantes de Sass est la possibilité d’imbriquer des règles. Dans un fichier CSS habituel, les règles (sélecteur/propriété/valeur) sont listées à la suite et chaque sélecteur doit inclure tous ses éléments

Avec l’imbrication, nous pouvons écrire un code SCSS qui est à la fois non redondant et plus facile à suivre :

//code CSS habituel
body.home .media-unit {
border: 1px solid #ccc;
background-color: #fff
}
body.home .media-unit .right {
border-left: 1px solid #ccc;
}
body.home .media-unit .right h1 {
font-size: 24px;
}

// code imbriqué
body.home{
.media-unit{
border: 1px solid #ccc;
background-color: #fff;
.right{
border-left: 1px solid #ccc;
h1{
font-size: 24px;
}
}
}
}

Le processeur transforme ensuite ce code en CSS normal, identique à la version précédente.
Malheureusement, la syntaxe SCSS plus courte ne réduira donc pas la taille de vos fichiers CSS. Mais l’imbrication vous aidera à avoir un code propre, logique et bien organisé, qui devrait être plus facile à entretenir au cours du temps.

Utiliser les variables en Sass

La déclaration de variables est très simple en Sass, la syntaxe est celle ci :

$maVariableCouleur: #be608b

Vous pouvez déclarer votre variable au début de votre ficher .scss ou dans un fichier .scss distinct, dédié à contenir toutes vos variables (comme nous le verrons au chapitre consacré aux imports.

Vous pouvez ensuite l'appeler n'importe où, et autant de fois que vous le voulez, dans votre SCSS de la manière suivante :

h1{
color: $maVariableCouleur;
}

Utiliser les partiels avec @import

La directive @import existe déjà dans CSS pour lier entre elles des feuilles de styles externes, et certains développeurs aiment l’utiliser pour organiser une feuille de style complexe en morceaux découpés logiquement. Par exemple, notre fichier style_principal.css pourrait ne contenir que des expressions @import liant des feuilles de style appliquées spécifiquement à chaque page d’un site.

Cependant, ce n’est pas considéré comme une bonne pratique en termes de performance, car chacun de ces imports est un fichier séparé, qui déclenche une requête et doit être chargé par le navigateur, ce qui peut ralentir le chargement de votre site

Sass est utile ici car il scinde les feuilles de style dans des partiels. On utilise là aussi la commande @import (sous une forme raccourcie), et quand vos fichiers SCSS sont processés, le contenu des partiels est inséré directement dans le CSS final.

// Syntaxe d'importation dans CSS
@import url(’/partage/global.css’);
@import url(’/pages/accueil.css’);
@import url(’/pages/blog.css’);

// Syntaxe d'importation dans SASS
@import ’/partage/global’;
@import ’/pages/accueil’;
@import ’/pages/blog’;

Cela donne un fichier CSS unique contenant tous vos styles, et donc une requête unique. Pour optimiser encore le temps de chargement, Sass peut aussi compacter automatiquement le fichier de sortie CSS en éliminant tous les espaces inutiles et les sauts de ligne.

Les mixins

Les Mixins sont des ensembles de propriétés ou de règles que vous pouvez inclure, ou “mixer”, dans d’autres règles. Vous les définissez en utilisant le mot clé @mixin et vous les intégrez en utilisant le mot clé @include.

Dans cet exemple, je demande à Sass d’appliquer toutes les propriétés contenues dans le mixin texte-surligne-gras à tous les éléments span situés à l’intérieur de .resultat-surligne.

Une fois défini le mixin, vous pouvez le réutiliser n’importe où dans le même fichier. Ici, je veux que les éléments de classe surligne aient également la couleur et le font weight spécifiés par le mixin.

// exemple de mixin
$couleur-surlignage: #ffa;
@mixin texte-surligne-gras {
font-weight: bold;
background-color: $couleur-surlignage;
}
.resultat-surligne {
span {
@include texte-surligne-gras;
}
}

//Exemple de réutilisation du mixin
.surligne {
@include texte-surligne-gras;
}
